# T038 — Redesign cleanup sweep: delete dead styling

## 2026-07-20

Removed the orphaned pre-redesign design system now that every screen has been
re-skinned (T016–T048):

- Deleted `HomeTileStyle`, `TileIcon`, `TileBackground`, `HomeSquareTile`, `HomeWideTile`,
  `HomeActionTile` from `Sources/Views/HomeView.swift` — confirmed unreferenced by grep first.
- Deleted the old `PlantPalette` 10-hue implementation from `Sources/Support/DisplayFormatting.swift`
  (kept `capitalisedWords`, which is unrelated). It was still referenced by two redesigned
  screens (`GuidedWateringView.swift`, `PlantDetailView.swift`) via `PlantThumbnail(tint:
  PlantPalette.color(for:))` — migrated both call sites to `PlantToken`/`PlantTokenPalette`
  (the design system PlantPalette was replaced by), which already had a `photo:` parameter
  covering the same "photo or tinted placeholder" behaviour. That made `PlantThumbnail` itself
  dead (nothing else referenced it), so it was deleted from `PlantListView.swift` too. Dropped
  the now-unused `import SwiftUI` from `DisplayFormatting.swift` and added `import UIKit` to the
  two files that now build a `UIImage` from `Data` inline.
- Converted the remaining raw `Image(systemName:)` SF Symbol stragglers to `ChromeIcon` in every
  already-redesigned screen that still had them: `AddFlowView.swift` (T022/T023 — house, chevron,
  plus, shuffle, magnifying glass, circle-plus), `PhotoPromptView.swift` (T024 — camera),
  `PhotoCaptureView.swift` (T025 — checkmark-circle, camera), `CapturePhotoView.swift` (T026 —
  camera), and `RoomInfoHeader.swift` (shared by the T029/T030/T046 room + edit-plant screens —
  info-circle). `grep -rn "Image(systemName:" Sources/Views` now returns nothing.
- Reviewed the `Color(red:green:blue:)` inline literals still under `Sources/Views` — every
  survivor is an actively-referenced one-off design token inline in a `.background()`/`.fill()`
  call (not leftovers of a deleted shared palette), so none were removed.
- Updated `Tests/DisplayFormattingTests.swift` to drop the `PlantPalette` test cases and its
  stale doc comment.
- Added a LIMITATIONS.md row: redesigned screens use fixed custom font sizes (`SproutFont`
  builds `Font.custom(name, size:)` with a literal point size, not `relativeTo:`), so none of
  them respond to Dynamic Type — a pre-existing gap now logged with its revisit trigger.

## Note (pre-existing, not touched by this task)

While spot-checking the detail screen, noticed `PlantDetailView.swift`'s `ChromeIcon.droplet`/
`ChromeIcon.pencil` icons (schedule-card zone 1) are missing `.resizable()` and spill past their
intended size — `ChromeIcon.image` wraps a bundled asset, not an SF Symbol, so `.font(.system
(size:))` doesn't constrain it (the same class of bug `FixedGlyphPlantToken`'s doc comment in
`AddFlowView.swift` calls out and works around elsewhere). This predates this task (unrelated to
`PlantPalette`/`PlantThumbnail`/`Image(systemName:)` cleanup) and is out of this task's scope —
flagging here rather than fixing inline.
