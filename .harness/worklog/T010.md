# T010 — SheetScaffold + SproutAlert

**Date:** 2026-07-17

## What was done

Implemented two core DesignSystem components per the redesign spec (§2.8–2.9):

### 1. `Sources/DesignSystem/SheetScaffold.swift`
- **SproutSheetHeader**: Reusable sheet chrome with:
  - Grab handle (40×5, rgba(60,66,58,0.2)) centered at top
  - Optional Cancel button (green, body 17)
  - Title in Bricolage display(18)
  - Optional Confirm button (green body 17 semibold)
  - Disabled state opacity (0.4)
- **`.sproutSheetBackground()` modifier**: applies paper background, radius 40, hides drag indicator
- Preview shows sheet header on paper background with standard styling

### 2. `Sources/DesignSystem/AlertCard.swift`
- **SproutAlert**: Full-screen confirmation dialog with:
  - Scrim background (rgba(25,40,30,0.4), black 0.4 opacity)
  - Centered 286pt card with paper/cream fill (rgba(244,241,231,0.92)) + ultraThinMaterial blur
  - Radius 22, dialog shadow
  - 52Ø tinted icon circle (tint at 0.13 opacity, icon full tint 20pt)
  - Title in Bricolage display(19)
  - Message in body(13.5) textMuted color, centered
  - Two equal buttons (radius 13):
    - Cancel: transparent with ink label
    - Confirm: filled with tint (destructive #C4553B for example), white label
  - Semantic support for destructive/neutral roles
- **`.sproutAlert(isPresented:) { }` modifier**: overlay at infinity, ignoring safe area, with fade+scale(0.95→1) transition

### 3. `Sources/DesignSystem/SheetAlertPreview.swift`
- Combined preview showing both components working together
- Sheet can be dismissed
- Alert overlays the sheet with destructive action ("Delete Basil?" example)
- Demonstrates proper visual hierarchy and stacking
