## What was done

### PlantEditView.swift — complete redesign
Replaced the old NavigationStack + Form layout with SproutSheetHeader chrome and redesigned sections:
1. **Header**: SproutSheetHeader with "Edit Plant" title, Cancel/Save buttons (Save disabled until nickname is non-empty)
2. **PHOTO section**: eyebrow label + white row with 64Ø PlantToken (photo-wins) + "Add photo" / "Change photo" green button
3. **ICON section**: eyebrow label + white row with 34Ø FixedGlyphPlantToken + "Change icon" green button → sheets to IconPickerView
4. **NICKNAME section**: eyebrow label + white text field
5. **ROOM section**: eyebrow label + white row with Menu room picker (shows current room or "None")
6. **DELETE section**: 30pt gap, white row with red border, trash icon + "Delete Plant" red semibold button → SproutAlert confirm dialog
7. Full-screen cover for camera (CapturePhotoView) on photo capture
8. SproutAlert for delete confirmation: "Delete {name}? This removes the plant and its check-in history. This can't be undone." with red Delete button

The form uses:
- SproutSheetHeader (header bar with Cancel/Save, T010)
- SectionEyebrow (section labels, T007)
- PlantToken + FixedGlyphPlantToken for visual display (T005)
- SproutAlert for the delete confirmation (T010)
- IconPickerView for icon selection (T021)
- Existing camera flow via CapturePhotoView

### PlantEditViewModel.swift — icon + delete support
- Added `@Published var plantIcon` to track the selected icon
- Added `@Published var plantID` to expose the plant's ID (needed for token colors + icon picker)
- Updated init to load icon and plantID from the plant in edit mode
- Updated save() to persist the icon alongside nickname/room
- Added plantForIconPicker() to construct a Plant for the icon picker with current form values
- Made repository public so PlantEditView can call delete() directly

### ContentView.swift — no changes needed
The makeEditor closure already passes the repository to PlantEditViewModel, which the view now uses for delete.
