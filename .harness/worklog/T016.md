# T016 — Home screen assembly (Botanical Editorial)

**2026-07-17**

## What I did

Rebuilt `HomeView`'s body in the new "Botanical Editorial" language, replacing the old
gradient-tile bento grid while preserving every deep link and existing behaviour:

- Custom top bar: `HomeLogoLockup` (34×34 r11 `logoGradient` tile + seedling + "Sprout"
  display 22) left; right `CircularToolbarButton`s (from the existing `DesignSystem/Cards.swift`)
  — bell-slash `#C4663F` only when reminders are off, gear `#4A5142`. Replaced the old
  `navigationTitle`/toolbar with `navigationBarHidden(true)` since the bar is now hand-drawn.
- Greeting eyebrow + headline sourced from `HomeHeroCard.greeting(for:hour:)` (T015).
- Restyled reminders-off row (`HomeRemindersOffBanner`: white r18, terracotta border/icon,
  same tap → `gatekeeper.enable()`), replacing the old `NotificationsOffBanner` (removed —
  fully superseded, not in the "leave in place" struct list).
- `HomeHeroCard` wired to `PlantListViewModel`'s due data: due plants → `.due`; no plants →
  `.empty`; otherwise `.allWatered` with the soonest-scheduled remaining plant. Since
  `PlantListViewModel.Item` doesn't retain the plant's `icon`/room fields (out of scope —
  `Sources/ViewModels/**`), reconstructed a display-only `Plant` per item using the species'
  default icon; the token still prefers a photo when one exists, so this only affects the
  (currently unused-by-demo-seed) icon fallback for plants without photos.
- Bento row: sage "My Plants" (token stack of up to 3 due-or-any plants, 38Ø, sage border,
  "{N} growing"/"None yet") → plant list; oat "Rooms" (house bubble, "{N} spaces"/"Set one
  up") → rooms. Added a second `RoomsViewModel` instance (via the already-injected
  `makeRooms` factory) purely to back the Rooms tile's live count.
- Ghost row: "+ Add a plant" / "☑ Check in" using the existing `SproutGhostButtonStyle`.
- Moved the now-dead tile structs (`HomeTileStyle`, `TileIcon`, `HomeSquareTile`,
  `HomeWideTile`, `HomeActionTile`, `TileBackground`) out of the body path but left the types
  in place per the task spec (cleanup task deletes them later).
- `HomeTileText.plantsSubtitle` updated to the new "{N} growing"/"None yet" copy; added
  `roomsSubtitle`. Left `waterSubtitle`/`checkInSubtitle`/`statusLine` untouched (unused by
  the new body, same "leave for the cleanup task" treatment as the dead structs).

## Bug found + fixed (in scope)

Wiring the hero's avatar stack + the My Plants token stack onto a real device surfaced a
pre-existing `PlantToken` (DesignSystem, out of scope) bug: its icon-fallback glyph isn't
`.resizable()`, so on a real bundled asset (not an SF Symbol) it renders at native size and
overflows across the whole screen instead of sitting inside the 36/38pt token. Added
`HomePlantAvatar` (in `HomeComponents.swift`, in scope) which delegates to `PlantToken` for
the photo case (already correct) but draws its own gradient + resizable glyph for the
icon-fallback case, then clips to the token's circle. Confirmed via before/after screenshots.
