## Overview
The guided-watering header title "Water your plants" wraps onto two lines ("Water your" / "plants") on the guided-watering screens. In Sources/Views/GuidedWateringView.swift the headerBar (lines 39-62) is a hand-rolled HStack of three items: a leading Button("Close") with .frame(maxWidth: .infinity, alignment: .leading) (lines 41-44), the centre Text("Water your plants").font(SproutFont.display(17, weight: .bold)) (lines 46-48), and a trailing Button("Skip") (or a Spacer when there is no current plant) also with .frame(maxWidth: .infinity, alignment: .trailing) (lines 50-58). Because BOTH flanking elements take maxWidth: .infinity they greedily expand and starve the middle Text of horizontal space, and the Text has NO .lineLimit(1) / .fixedSize(horizontal:) / .layoutPriority, so it wraps. The font size (17) is fine — this is width starvation, not an oversized font. The same header chrome is reused across the guided-watering report/action/done steps, so the fix must keep those balanced (Close pinned left, Skip pinned right, title centred).
