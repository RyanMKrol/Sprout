# T003 — SproutTheme design tokens

## Overview
Create the single source of styling truth for the redesign: every color, gradient, radius,
shadow and spacing constant from the design, as named constants. Later tasks bind to these
names and never re-derive hex values.

## Do
1. Read `.harness/docs/designs/redesign-spec.md` §1.2 (palette) + §1.3 (geometry/shadows) and
   `.harness/docs/designs/redesign-architecture.md` §2 (SproutTheme contract — the exact member
   names to create).
2. Create `Sources/DesignSystem/SproutTheme.swift` with `enum SproutTheme` holding static
   `Color`/gradient members for every §1.2 value (paper #F4F1E7, ink #232821, brandGreen #2F6B4C,
   heroGradient 150° #2F6B4C→#193E2C, logoGradient 150° #3C7E58→#1E4632, launchGradient 165°
   #2F6B4C→#173726, the text ramp, chrome colors, semantic colors, sage/oat surface sets — all of
   them, exactly the architecture contract's names). Add a private `Color(hex:)` init helper.
3. Add `SproutTheme.Radius` (sheet 40, hero 28, bento 24, dialog 22, row 20, field 18, button 16,
   segmented 15, pill 11, chip 10) and shadow ViewModifier helpers (cardShadow, bentoShadow,
   heroShadow, primaryButtonShadow, dialogShadow) per spec §1.3.
4. Add `Tests/SproutThemeTests.swift` asserting: `Radius` values match the list above, and
   spot-check 3 colors resolve to the expected sRGB components (e.g. paper ≈ (0.957,0.945,0.906)
   within 0.01).

## Done when
- `xcodebuild test` passes including `SproutThemeTests`.
- Every color/radius/shadow named in redesign-architecture.md §2's SproutTheme comment exists;
  no screen files changed (tokens only).
