# T039 — Visual fidelity review (human gate)

## Overview
Owner walkthrough of the finished redesign against the design canvas before the wave is called
done — taste calls a machine can't make.

## Do
1. Build and run the app seeded (`./build_run.sh Sprout-Claude -seedDemoData YES`) and walk
   every screen against `docs/designs/app-redesign/Sprout App.dc.html` (open in a browser) and
   `.harness/docs/designs/redesign-spec.md` §3 — including the ⚑ derived states (all-watered
   hero, outcome variants, room delete, Other-room path, icon-picker entries).
2. File anything off as ideas (`/implementation-harness-capture-idea`) or follow-up tasks;
   tweak derived copy you dislike directly or via tasks.
3. When satisfied, mark this task done: `.harness/scripts/mark-done.sh T039`.

## Done when
- The owner has walked every redesigned screen and either accepted it or filed follow-ups, and
  marked this task done via mark-done.sh.
