# T001 — Harness bootstrap on the existing app — CI green on main

## Overview

Sprout's v1 was fully built by an earlier (pre-plugin) harness, so unlike a greenfield T001 there
is no project scaffold to create — the app, tests, `project.yml`, and `build_run.sh` all exist and
pass locally. What this task bootstraps is the **new** harness's merge gate: this repo has never
had GitHub Actions CI before (the old harness was local-only, `REQUIRE_CI=0`). The one-time human
steps are proving the new CI workflow green on `main` and standing up the dedicated local
simulator, so every later loop-built task has a working gate to merge through.

This is a 🔒 needs-human gate task: the loop skips it and records `failed:blocked` until a human
marks it done.

## Do

1. Commit and push the scaffolded harness (`.harness/`, `.github/`, `.swiftlint.yml`,
   `tools/loop_sim.sh`, `CLAUDE.md`, `.gitignore`) to `main`.
2. Create the dedicated simulator once locally: `./tools/loop_sim.sh` (creates **Sprout-Claude**
   on the newest iOS runtime and prints its UDID; idempotent).
3. Run the full LOCAL_DOD once end-to-end on this machine and confirm it passes:
   `./tools/loop_sim.sh >/dev/null && xcodegen generate && swiftlint lint && xcodebuild test
   -project Sprout.xcodeproj -scheme Sprout -destination 'platform=iOS Simulator,name=Sprout-Claude'`
4. Watch the **CI** workflow run for the push on `main` (macos runner: swiftlint → select
   Xcode 26 → xcodegen → resolve simulator → `xcodebuild test`) and get it green. Expected
   first-run risks: runner Xcode/simulator availability (the composite actions handle selection),
   and lint errors (swiftlint must already pass locally before pushing).
5. Confirm the `lint-workflows` (actionlint) workflow is also green.
6. Mark this task done: `.harness/scripts/mark-done.sh T001`.

## Done when

- The `CI` workflow (name matching `CI_WORKFLOW` in `.harness/config/harness.env`) is green on
  `main` for the harness-scaffold commit.
- `lint-workflows` is green.
- The **Sprout-Claude** simulator exists locally and the LOCAL_DOD command passes end-to-end.
- `DRY_RUN=1 .harness/scripts/loop.sh` runs cleanly and shows the backlog.
- T001 is marked done via `mark-done.sh` (never by hand-editing `TASKS.json`).
