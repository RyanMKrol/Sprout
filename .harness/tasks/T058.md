## Overview
In the Add Room sheet (Sources/Views/AddRoomView.swift), picking a preset room type (Living Room, Kitchen, Bedroom, …) currently shows only a read-only 'TYPICAL SETTINGS' card with two static chips — there is no way to adjust the light/humidity for that room. The editable Direct Sun / Indirect Sun / Humidity segmented controls appear ONLY on the 'Other…' custom path. Because rooms of the same type vary a lot (some living rooms are brighter or more humid than others), the user should be able to fine-tune those levers up front instead of saving and then editing the room. The machinery already exists: AddRoomView already keeps @State directSun/indirectSun/humidity (lines 17-19) and already adopts the selected preset's values into that state (applyPresetDefaults, ~lines 135-140, fired from .onChange(of: selection)), so Save already threads the (possibly overridden) values through. The exact editable lever UI (Direct Sun / Indirect Sun / Humidity segmented Pickers, each with a RoomInfoHeader ⓘ info header) already exists in AddRoomView's customControlsSection (~lines 216-258) and in RoomEditorView (Sources/Views/RoomsView.swift:121). This task surfaces those same editable controls on the preset path, pre-filled from the preset. NOTE: the replace-vs-augment decision (whether the editable controls REPLACE the read-only 'TYPICAL SETTINGS' chips or are shown IN ADDITION to them) is being confirmed with the owner via a pending question — follow that answer; if it is still unresolved when building, prefer REPLACING the read-only chip card with the editable controls (simpler, no duplicated humidity value).
