## Overview

Make the launch-screen background colour durable. T041 shipped the animated splash and wired the
`LaunchBrandGreen` colour into the `UILaunchScreen` background, but it wrote that only into the
**generated** `Sources/Info.plist` — not into `project.yml`, which is the XcodeGen source of truth.
Because `xcodegen generate` runs on every build and regenerates `Info.plist` from `project.yml`
(whose `UILaunchScreen:` is currently an empty `{}`), the colour is stripped on every regen and the
pre-SwiftUI launch frame reverts to the default background. This task moves the setting into the
source so it survives regeneration.

## Do

In `project.yml`, under the target's `info.properties.UILaunchScreen` key (currently `UILaunchScreen: {}`,
around line 39), set the launch-screen background to the existing colour asset — i.e. give it a
`UIColorName: LaunchBrandGreen` entry so the generated plist keeps the colour. The `LaunchBrandGreen`
colour asset already exists at `Sources/Assets.xcassets/LaunchBrandGreen.colorset/` — do NOT create a
new asset; just reference it by name. Then run `xcodegen generate` so the regenerated
`Sources/Info.plist` reflects the change, and commit the regenerated `Info.plist` alongside the
`project.yml` edit so the tree is consistent. Do not change anything else in `project.yml` or the plist.

## Done when

- `project.yml`'s `UILaunchScreen` block references the colour — it contains `UIColorName: LaunchBrandGreen`
  (no longer an empty `{}`).
- After running `xcodegen generate`, `Sources/Info.plist`'s `UILaunchScreen` dict contains
  `<key>UIColorName</key><string>LaunchBrandGreen</string>` (verify e.g. with
  `plutil -extract UILaunchScreen.UIColorName raw Sources/Info.plist` printing `LaunchBrandGreen`).
- Running `xcodegen generate` a second time produces **no** change to `Sources/Info.plist` — i.e.
  `git status --porcelain Sources/Info.plist` is empty after a regen (the colour no longer drifts away).
- The build still succeeds (`./build_run.sh Sprout-Claude`) and the app launches normally.
