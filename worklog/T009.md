## 2026-06-01 — attempt 1 (done)

Fresh start; clean `t009` branch off `main` (no prior partial work).

### What I did
- Added `Sources/Engine/ScheduleEngine.swift`:
  - `Clock` protocol with `SystemClock` (production) and `FixedClock` (tests) so "now"
    is injected — the engine never reads the wall clock.
  - `ScheduleEngine` (pure): `effectiveInterval(for:adj:weatherFactor:)` implements
    `clamp(round(baseIntervalDays × weatherFactor × adj), min, max)`, clamping `adj` to
    `Plant.adjRange` ([0.5, 2.0]) first. `weatherFactor` defaults to `1.0`.
  - Two `nextDue` overloads: one anchored at an explicit `lastWatered` (for T010/T011),
    one taking a `Plant` + `Clock` that anchors at `clock.now` when never watered.
  - `calendar` injectable (default `.current`) for deterministic day arithmetic.
- Added `Tests/ScheduleEngineTests.swift` — 16 tests using a UTC calendar + `FixedClock`:
  base case, adj scaling, rounding (half-away-from-zero boundary cases), min/max clamping
  across all three moisture species, adj-out-of-range clamping (below/above), weather
  factor (hot shortens / cold lengthens / composes with adj), next-due date arithmetic
  (from lastWatered, plant-with/without lastWatered, adj+weather), and determinism.
