# T208 — Wire post-create photo flow

## 2026-06-01 — done

- `Sources/Views/PlantListView.swift`:
  - The basket sheet now stashes the created plants (`pendingPhotoPlants`) and uses the sheet's
    `onDismiss` (`offerPhotosIfJustCreated`) to refresh the list and — if plants were created —
    raise a "Take a photo of each new plant?" `confirmationDialog`. Running it in `onDismiss`

    existing `.fullScreenCover` (T207) over them in basket order; "Not Now" clears and returns to
    the refreshed list. Finishing the camera reloads the list.
  - Added a `SPROUT_SCREEN=photoprompt` deep-link for the screenshot.

This reuses the T207 presentation infrastructure (`makePhotoCapture`, `photoTargets`,
`.fullScreenCover`); T208 only adds the prompt + the created→targets hand-off.
