# T212 — Drive the schedule from rooms; retire weather

## 2026-06-01 — done

- **Per-plant environment factor** replaces the global weather factor:
  - `ContentView`: removed `weatherFactor` `@State`, `.task`, `refreshWeatherFactor`, `initialWeatherFactor`;
    added `static environmentFactor(for:repository:)` (plant.roomID → room → `RoomEnvironment.factor`,

    tests stay valid).

  + `TemperatureUnit`); `SettingsView`/`SettingsViewModel` drop those controls; the weather/GPS path is no
  longer called. `Sources/Weather/*` + `WeatherFactor` left dormant (LIMITATIONS row).
- Tests updated: ScheduleExplanationTests (room causes), SettingsViewModelTests (removed unit/weather tests),
  WeatherFactorTests (explanation assertion → driesFaster), new BasketAddViewModel initial-cadence tests.
