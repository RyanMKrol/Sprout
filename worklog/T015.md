# T015 — Weather provider (Open-Meteo + CoreLocation)

## 2026-06-01

**Did**
- Added `Sources/Weather/`:
  - `WeatherForecast.swift` — source-agnostic `Equatable` forecast value (`days: [Day]` with
    max/min °C + precipitation mm) plus `init(openMeteoData:)` decoding Open-Meteo's
    column-major `daily` arrays into row-major `Day`s (truncates to the shortest series).
  - `WeatherProviding.swift` — the `WeatherProviding` protocol + `OpenMeteoWeatherProvider`
    (key-less Open-Meteo `/v1/forecast`, injectable `URLSession`, requests only
    `temperature_2m_max/min,precipitation_sum`).
  - `LocationProviding.swift` — `Coordinate`, the `LocationProviding` protocol (returns `nil`
    for every can't-locate outcome), and `CoreLocationProvider` wrapping `CLLocationManager`
    (when-in-use auth, one km-accuracy fix, continuation resumed exactly once; all failure
    paths → `nil`).
  - `WeatherFactorService.swift` — ties location + forecast into `currentWeatherFactor()`,
    falling back to `ScheduleEngine.defaultWeatherFactor` (1.0) when location is `nil` or the
    fetch throws. `factor(for:)` is the neutral T015 placeholder; **T016** replaces it.
- Added `Tests/Fixtures/openmeteo_london.json` (real-shaped 7-day response) and
  `Tests/WeatherProviderTests.swift` (6 tests): fixture decode, short-series truncation,
  garbage-throws, denied-location → 1.0, forecast-failure → 1.0, available → surfaces forecast.
  Stub `LocationProviding`/`WeatherProviding` — **no network**.
- `project.yml`: added `NSLocationWhenInUseUsageDescription` (the pre-existing `# T015 adds …`
  placeholder). Recorded the out-of-scope manifest line, the neutral-factor placeholder, and the
  untested `CoreLocationProvider` in `docs/LIMITATIONS.md`.
