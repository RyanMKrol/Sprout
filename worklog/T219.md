# T219 — Edit flow: change a plant's photo

## 2026-06-02

**What I did.** Added a photo path to the narrowed Edit Plant form (T218), reusing the
existing single-shot `PhotoCapturing` seam (T207) and `PlantPhoto.encode`.

- `Sources/ViewModels/PlantEditViewModel.swift`: added a `camera: PhotoCapturing`
  dependency, `@Published private(set) var photoData` (pre-filled from the plant in edit
  mode), `isCapturingPhoto`, `hasPhoto`, and an async `changePhoto()` that captures →
  `PlantPhoto.encode` → stages the bytes in-memory. `save()` now writes `photoData` onto the
  plant (both edit and add branches). Transactional: Cancel discards a just-captured photo.
- `Sources/Views/PlantEditView.swift`: new **Photo** section showing the current photo (or
  the shared `PlantThumbnail` leaf placeholder) and an "Add photo"/"Change photo" button that
  runs `changePhoto()` in a `Task`, with a `ProgressView` while capturing.
- `Sources/ContentView.swift`: `makeEditor` now passes `makeCamera()` (stub on
  simulator/DEBUG, `AVFoundationCamera` on device) into the view model.
- `Tests/PlantEditViewModelTests.swift`: added 5 tests — prefill existing photo, no-photo
  case, capture+persist-on-save (staged before save, persisted after), replace existing, and
  failed-capture keeps the current photo. Helper builds the stub in-body (the `@MainActor`
  stub can't be a default arg in a nonisolated context).
