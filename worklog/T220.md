# T220 — Room light model: direct/indirect sliders → inferred brightness + tooltips

## 2026-06-02 — implemented (done)

### What I did
- **`Sources/Model/Room.swift`**: added `LightLevel` (low/medium/high) and `Brightness`
  (dark/medium/bright) enums. `Brightness.inferred(directSun:indirectSun:)` weights direct sun
  twice as heavily as indirect (score 0…6 → dark 0–1, medium 2–3, bright 4–6), so it's monotonic
  in both inputs. `Room` now carries `directSun` + `indirectSun` (+ a derived `brightness` and a
  `Bright · Dry` `environmentSummary`). Kept the legacy `sunlight` field, defaulted/derived, so
  existing callers/stores still work. Two inits: a primary (directSun/indirectSun) and the legacy
  (`sunlight:`) one that derives the two levels — `Room(name:)` still resolves to the legacy init.
- **`Sources/Engine/RoomEnvironment.swift`**: `factor` now consumes `Brightness` × humidity
  (`factor(brightness:humidity:)`, `factor(directSun:indirectSun:humidity:)`, `factor(for:)`),
  clamped to the same ±30% band. Removed the old `factor(sunlight:humidity:)`.
- **`Sources/Persistence/StoredModels.swift`**: added **optional** `directSun`/`indirectSun` to
  `StoredRoom` (additive, no migration). `toDomain()` falls back to values derived from the legacy
  `sunlight` for pre-T220 rows (nil light levels). Updated `init(domain:)` + `applyScalars`.
- **`Sources/Views/RoomsView.swift`**: room editor now has two segmented light controls (Direct
  sun / Indirect sun) each with a small ⓘ info button (`InfoHeader` → popover), a live "Inferred
  brightness" row + explanatory footer, and humidity unchanged. Added a `SPROUT_SCREEN=roomeditor`
  deep-link (DEBUG) that auto-opens the editor for the first room.
- **`Sources/ViewModels/RoomsViewModel.swift`** (out of declared scope, additive): added
  `add(name:directSun:indirectSun:humidity:)` overload to wire the editor's add path without the
  data loss of round-tripping through the 3-value legacy enum. Kept the legacy `add(...:sunlight:...)`.
- **`Sources/Views/HomeView.swift`** (out of declared scope, 1 line): route `roomeditor` to push
  Rooms (which then auto-opens its editor) for the screenshot.
- **Tests**: rewrote `Tests/RoomEnvironmentTests.swift` to cover brightness inference (direct
  dominance, monotonicity, clamp band, extreme combos). Added round-trip tests for the new light
  fields in `Tests/PersistenceTests.swift`.
- Docs: TASKS.md box ticked, README.md row → ✅, `docs/LIMITATIONS.md` row added.
