# T223 — Fix multi-add photo prompt (decline + connected presentation)

## 2026-06-02

### What I did
Replaced the bare `confirmationDialog` "Take a photo of each new plant?" prompt — which
floated, disconnected, over the (possibly long) plant list — with a **connected sheet**.

- **New `Sources/Views/PhotoPromptView.swift`** (reusable):
  - A proper sheet titled "New plants added", anchored to the add flow.
  - Camera badge + heading "Add photos of your new plant(s)?" + a subtitle that always
    mentions skipping.
  - A "New plants (N)" section listing the just-created plants (name + species) so the
    prompt reads as part of the add flow, not a detached dialog.
  - A prominent **Take Photos** primary button and an equally obvious **Skip photos**
    decline, plus a "Skip" cancel in the nav bar.
  - Pure `PhotoPromptText` enum for the singular/plural copy (unit-testable).
- **`HomeView.swift` + `PlantListView.swift`**: swapped each `.confirmationDialog` for a
  `.sheet` presenting `PhotoPromptView`. The camera launches from the prompt sheet's
  `onDismiss` (new `startPhotosOnDismiss` flag) to avoid a present-while-dismissing race.
  Replaced `pendingPhotoPlants: [Plant]` with `promptTargets: [PhotoCaptureCoordinator.Target]`
  driving both the prompt display and the camera launch.
- **`PlantListView` deep-link**: `SPROUT_SCREEN=photoprompt` now seeds `promptTargets` from
  the seeded plants so the screenshot shows the connected sheet with real content.
- **New `Tests/PhotoPromptTextTests.swift`**: covers the prompt copy (singular/plural title,
  skip-mention subtitle, list header).
