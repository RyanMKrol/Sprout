# T014 — Settings — preferred reminder time, units, weather toggle

## 2026-06-01 — built (done)

**What I did**
- Added `Sources/ViewModels/SettingsViewModel.swift`:
  - `TemperatureUnit` enum (`.celsius`/`.fahrenheit`, with `symbol`/`label`).
  - `AppSettings` — `Codable`/`Equatable` value type (`reminderHour`, `temperatureUnit`,
    `weatherEnabled`) with `.default` (reminder hour = `WateringNotificationScheduler.defaultReminderHour`
    = 9, °C, weather on) and `reminderHourRange` (0…23).
  - `SettingsStore` protocol + `UserDefaultsSettingsStore` (JSON under one key) — the
    persistence boundary the VM depends on, so tests inject ephemeral defaults.
  - `SettingsViewModel` (`@MainActor`, `ObservableObject`): reads the store on init; mutating
    methods `updateReminderTime(_:)`/`updateReminderHour(_:)` (clamped, persist, **reschedule
    every plant's reminder** at the new hour via an injected `makeScheduler` factory +
    repository), and `setTemperatureUnit`/`setWeatherEnabled` (persist only). `reminderTime`
    projects the hour back to a `Date` for the picker.
- Added `Sources/Views/SettingsView.swift` — a `Form` with a `DatePicker` (hour-and-minute)
  for reminder time, a Temperature segmented-style picker, and an "Adjust for weather" toggle,
  each bound through the view model; "Done" dismisses.
- Wired the entry point (minimal edits outside the Settings scope, same precedent as
  T007/T008/T011): `PlantListView` gained a gear toolbar button (`makeSettings` factory) + a
  `.sheet(isPresented:)` and a `SPROUT_SCREEN=settings` deep-link case; `ContentView` builds
  `makeSettings` against the shared repository + `UserDefaultsSettingsStore`.
- Added `Tests/SettingsViewModelTests.swift` — 7 tests: default settings on a fresh store;
  **persistence across launches** (a second VM over the same defaults reads back the saved
  values); **reminder-time change reschedules at the new hour** (real `WateringNotificationScheduler`
  driven against a recording stub center — trigger lands on the due day at the chosen hour);
  hour clamping; reschedules *every* plant; `reminderTime` projection; unit/weather persist
  independently without touching the reminder hour.
