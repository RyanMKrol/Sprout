# T216 — Developer settings: delete all plants & rooms

## 2026-06-01

**Did:**
- Added `deleteAllPlants()` and `deleteAllRooms()` to the `PlantRepository` protocol.
  - `SwiftDataPlantRepository`: `deleteAllPlants()` deletes every `StoredPlant` (check-ins
    cascade); `deleteAllRooms()` first nils `roomID` on any assigned plants (plants are kept),
    then deletes every `StoredRoom`. Both no-op on an empty store.
  - `EmptyPlantRepository` (ContentView): added the two no-op stubs.
- `SettingsViewModel`: added an injected `onDataReset` callback (default no-op) and a
  `deleteAllData()` method that wipes plants + rooms via the repository, then calls
  `onDataReset()` so the host refreshes. Degrades to a no-op on repo errors / nil repo.
- `SettingsView`: added a **Developer** section with a destructive "Delete all plants & rooms"
  button behind a `confirmationDialog` ("Delete everything" / Cancel); confirming calls
  `deleteAllData()` then dismisses.
- `ContentView.makeSettings`: wired `onDataReset: { listViewModel.load() }` so the home tiles
  + My Plants list refresh to their empty state after a reset.

**Tests (added):**
- `PersistenceTests`: `testDeleteAllPlantsAndRoomsEmptiesStore`,
  `testDeleteAllRoomsKeepsPlantsButClearsRoomID`, `testDeleteAllOnEmptyStoreIsNoOp`.
- `SettingsViewModelTests`: `testDeleteAllDataWipesPlantsAndRoomsAndNotifies`,
  `testDeleteAllDataWithoutRepositoryStillNotifies`.
