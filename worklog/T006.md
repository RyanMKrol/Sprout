# T006 — My Plants list + empty state

## 2026-06-01 — built, verified, ready to integrate

**What I did**
- Added `Sources/ViewModels/PlantListViewModel.swift`:
  - `DueStatus` — a pure value type (`unscheduled` / `overdue(days:)` / `dueToday` /
    `due(days:)`) classified from a plant's `nextDue` vs. an injected `now` by **calendar
    days** (start-of-day), with a `.label` for the pill and a `needsWater` flag. Decision is
    asserted in tests, not a brittle string.
  - `PlantListViewModel` (`@MainActor`, `ObservableObject`) — `load(now:)` reads the
    `PlantRepository` and projects plants into `Item`s ordered **soonest-due first**
    (`ordered(_:)`: earliest `nextDue`, unscheduled last, nickname tie-break). Repo errors
    degrade to an empty list. `isEmpty` drives the empty state.
- Added `Sources/Views/PlantListView.swift` — `NavigationStack` + `List` of `PlantCardView`
  (water-drop indicator colour-coded by urgency, nickname/species, colour-coded next-due pill),
  and `PlantListEmptyState` (`ContentUnavailableView`) for first-run.
- Wired the app to the real model: `DemoSeed` now exposes real `[Plant]` + a seeded in-memory
  `PlantRepository` (`seededRepository()`), replacing the throwaway `DemoPlant`/`DemoPlantListView`.
  `ContentView` builds a `PlantListViewModel` from the seeded in-memory store under
  `-seedDemoData YES`, else `PlantStore.persistent()`, with a safe empty fallback. Same
  `-seedDemoData YES` / `SPROUT_SCREEN` contract preserved.
- Tests: `Tests/PlantListViewModelTests.swift` — empty→empty-state, due-order (incl. unscheduled
  last + nickname tie-break), `DueStatus` classification (incl. time-of-day ignored) and labels.
