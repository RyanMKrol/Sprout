### IMPORTANT finding ‚Äî stale incremental build caused false-blank screenshots
Early captures were 100% white even though `** BUILD SUCCEEDED **`. Root cause was **not** the
code: `build_run.sh`'s incremental `xcodebuild build` + `simctl install` left the simulator
running an **old binary** (a `Color.red` full-screen diagnostic also rendered white; `strings`
on the *installed* binary lacked the new marker). Fix that made rendering work:
`rm -rf build ~/Library/Developer/Xcode/DerivedData/Sprout-*` then re-run `build_run.sh`.
Two compounding capture gotchas: `simctl io ‚Ä¶ screenshot` returns a 6√ó9 stub unless the
**Simulator GUI window is open**, and a mid-launch capture shows the white launch screen ‚Äî so
capture ~7 s after launch with the Simulator open. This is the same reason T001's shot was blank.
All of this is now documented in `CLAUDE.md` + `docs/LIMITATIONS.md` so future UI tasks don't
re-derive it. (`build_run.sh` itself is out of T002 scope, so it was left unchanged; a `clean`
step there is suggested in LIMITATIONS if the pitfall recurs.)
